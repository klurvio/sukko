# Plan: Reorganize Package Structure & Extract Shared Code

**Status:** ✅ Implemented

## Problem Statement

The current `internal/` directory mixes service-specific code with shared utilities, making it hard to understand what belongs where:

1. **Metric pollution**: Gateway and provisioning import `monitoring` for `NewLogger()` and `RecoverPanic()`, but `init()` registers 37 unused `ws_*` server metrics
2. **Inconsistent patterns**: `monitoring/metrics.go` uses manual `prometheus.MustRegister()` while gateway/provisioning use `promauto`
3. **No alerting in gateway/provisioning**: Only server has alerting capability
4. **No audit logging in gateway/provisioning**: Cannot track auth decisions, API calls
5. **Tight coupling**: Cannot use logging without pulling in metrics
6. **Unclear organization**: `internal/` mixes service-specific (`server/`, `gateway/`) with shared code (`auth/`, `kafka/`)
7. **Duplicate metric collection**: Three places collect system metrics (MetricsCollector, Server.collectMetrics, SystemMonitor)

## Goals

- [ ] Reorganize `internal/` with clear `shared/` subdirectory
- [ ] Extract shared logging utilities to `pkg/logging`
- [ ] Extract shared alerting to `pkg/alerting`
- [ ] Extract shared audit logging to `pkg/audit`
- [ ] Consolidate server metrics into `internal/server/metrics/` directory
- [ ] Add rate limiting to alerting (prevent Slack spam)
- [ ] Integrate alerting and audit logging into gateway and provisioning
- [ ] Delete `internal/monitoring/` package entirely
- [ ] Eliminate duplicate metric collection

## Package Dependency Graph

```
pkg/logging     pkg/alerting     pkg/metrics
     │               │                │
     │               ▼                │
     │          pkg/audit             │
     │               │                │
     ▼               ▼                ▼
┌─────────────────────────────────────────────┐
│            internal/shared/                 │
│  (auth, kafka, platform, orchestration...)  │
└─────────────────────────────────────────────┘
                    │
        ┌───────────┼───────────┐
        ▼           ▼           ▼
   internal/    internal/   internal/
    server/      gateway/   provisioning/
```

Key design decisions:
- `pkg/*` packages have no dependencies on `internal/*`
- `pkg/alerting` has NO dependencies on other `pkg/*` packages
- `pkg/audit` depends only on `pkg/alerting`
- `pkg/logging` is independent
- `internal/shared/` contains code used by multiple services
- Each service owns its metrics in `{service}/metrics/`
